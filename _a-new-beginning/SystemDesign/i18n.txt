
INTERNATIONALISATION & LOCALISATION::
You only do i18n once (architectural effort). You do l10n every time you target a new country or language (content effort).
i18n creates the empty containers and the rules. It's the structure. 

i18n: 
    <h1>{i18n.t('greeting')}</h1>
localisation:
    i18n Key (Code)	l10n Value (Data in fr-FR.json)
    greeting	    Bienvenue !

EXAMPLE:

// --- 1. Localization Data (The Resource Files - l10n) ---
// This data would typically be loaded from external JSON files in a real app.
const translations = {
    // English (United States)
    'en-US': {
        'welcome_greeting': 'Welcome to our platform!',
        'language_name': 'English',
        'switch_button': 'Switch to Hindi (hi-IN)'
    },
    // Hindi (India)
    'hi-IN': {
        'welcome_greeting': 'हमारे मंच में आपका स्वागत है!', 
        'language_name': 'हिन्दी',
        'switch_button': 'Switch to English (en-US)'
    },
    // Default Fallback
    'default': {
        'welcome_greeting': 'Welcome! (Default)',
        'language_name': 'Default',
        'switch_button': 'Switch Language'
    }
};

// --- 2. Internationalization Function (The Retrieval Logic - i18n) ---
// This core function handles string lookup and fallbacks.
function t(key, locale) {
    if (translations[locale] && translations[locale][key]) {
        return translations[locale][key];
    }
    // Fallback logic
    return translations['default'][key] || `Missing key: ${key}`;
}

// --- 3. The React Component ---

const App = () => {
    // State to manage the current locale setting
    const [locale, setLocale] = useState('en-US');

    // Function to toggle the locale between English and Hindi
    const toggleLocale = () => {
        setLocale(prevLocale => (prevLocale === 'en-US' ? 'hi-IN' : 'en-US'));
    };
    
    // Determine the next locale for the button text (l10n)
    const nextLocale = locale === 'en-US' ? 'hi-IN' : 'en-US';

    // Helper function for button text using the current locale
    const getButtonText = () => {
        // We use the NEXT locale key to show what the button *does*
        return t('switch_button', locale);
    }

    return (
        <div className="min-h-screen bg-gray-50 p-8 flex flex-col items-center justify-center font-inter">
            <script src="https://cdn.tailwindcss.com"></script>
            <style>{`.font-inter { font-family: 'Inter', sans-serif; }`}</style>

            <div className="bg-white p-8 md:p-12 rounded-2xl shadow-xl max-w-lg w-full text-center">
                <header className="mb-6">
                    <h1 className="text-4xl font-extrabold text-indigo-700 mb-2">
                        {t('welcome_greeting', locale)}
                    </h1>
                    <p className="text-lg text-gray-600">
                        Current Locale: <span className="font-semibold text-indigo-500">{locale}</span>
                    </p>
                    <p className="text-md text-gray-500">
                        Language Name: {t('language_name', locale)}
                    </p>
                </header>

                <button
                    onClick={toggleLocale}
                    className="mt-6 w-full py-3 px-6 bg-indigo-600 text-white font-bold rounded-lg 
                               shadow-md hover:bg-indigo-700 transition duration-300 
                               focus:outline-none focus:ring-4 focus:ring-indigo-300 active:bg-indigo-800"
                >
                    {getButtonText()}
                </button>
            </div>
            
            <div className="mt-8 p-4 bg-yellow-50 border-l-4 border-yellow-400 rounded-lg text-sm text-yellow-800 max-w-lg w-full">
                <p className="font-bold">Key Distinction:</p>
                <p>The code (`App` component and the `t` function) is the **i18n** layer and stays the same. The data in the `translations` object is the **l10n** layer and is what changes.</p>
            </div>
        </div>
    );
};

export default App;



Internationalization (i18n) and Localization (l10n) are two distinct but interconnected processes essential for adapting a product for global markets.
Internationalization is the design and development process of preparing your product to handle multiple languages and regional conventions without making code changes for every new region. It is a one-time effort done by developers.
While Localization is the process of adapting the internationalized product for a specific region or market. It is an ongoing effort managed by translators and product teams.

1. Externalize Strings in some json file
2. Using UTF-8 (Unicode) everywhere. Ensuring the application stack (database, APIs, and frontend) supports the Unicode standard for handling diverse character sets (e.g., Chinese, Arabic, Cyrillic).
3. Designing UI components (like buttons and navigation) that can dynamically resize to accommodate longer strings (e.g., German words are often much longer than English).

eg:
// i18n provides the API, l10n provides the locale string
